# lawpowers — monorepo

Монорепо юридичних плагінів для Claude Code. Один маркетплейс `lawpowers` містить кілька плагінів-юрисдикцій. Наразі:

| Плагін | Юрисдикція | Префікс команд | Мова документації |
|---|---|---|---|
| [`ua`](./ua) | Україна | `/ua:…` | українська |
| [`pl`](./pl) | Польща | `/pl:…` | polski |

Плагіни незалежні — користувачі встановлюють лише потрібні (або обидва).

## Структура репо

```
lawpowers/                         # репо на GitHub: crankshift/lawpowers
├── CLAUDE.md                       # цей файл — monorepo-рівень
├── README.md                       # user-facing: установка обох плагінів
├── CHANGELOG.md                    # історія релізів маркетплейсу і плагінів
├── .version-bump.json              # файли, де тримаються версії
├── LICENSE                         # MIT — покриває все репо
├── .claude-plugin/
│   └── marketplace.json            # каталог маркетплейсу lawpowers
├── ua/                             # плагін "ua" — українське право
│   ├── CLAUDE.md                   # контекст для розробки UA-плагіна
│   ├── .claude-plugin/plugin.json  # name: "ua"
│   ├── agents/
│   └── skills/
└── pl/                             # плагін "pl" — польське право
    ├── CLAUDE.md
    ├── .claude-plugin/plugin.json  # name: "pl"
    ├── agents/
    └── skills/
```

## Принципи

- **Одна юрисдикція — один плагін.** Не змішувати UA і PL нормативку в одних агентах/скілах.
- **Мова плагіна — мова юрисдикції.** UA-плагін — українською; PL-плагін — польською.
- **Префікси команд.** `name` у `plugin.json` використовується як неймспейс — `/ua:…`, `/pl:…`. Файли агентів/скілів всередині плагіна не потребують префікса.
- **Спільна ліцензія.** MIT на всьому репо.
- **Незалежні версії плагінів.** Плагін `ua` і плагін `pl` мають власні версії в `plugin.json`. Маркетплейс має свою версію в `marketplace.json:metadata.version`.
- **Тег-формат:** `v<marketplace-version>` для загального релізу маркетплейсу; за потреби можна додавати тег `ua/vX.Y.Z` або `pl/vX.Y.Z` для окремих плагінів — поки єдиний monorepo-тег.

## Як додати новий плагін (наприклад, для іншої країни)

1. Створити директорію в корені (`/xx/`) з коротким ISO-кодом країни або юрисдикції.
2. Додати структуру:
   ```
   xx/
   ├── .claude-plugin/plugin.json     # name: "xx"
   ├── agents/
   ├── skills/
   └── CLAUDE.md
   ```
3. Додати новий запис у `marketplace.json` → `plugins[…]` з `"source": "./xx"`.
4. Додати запис у CHANGELOG.
5. Bump-нути версію маркетплейсу в `.claude-plugin/marketplace.json:metadata.version`.
6. Відкрити PR, мержити.
7. Створити тег і Release.

## Плагін-специфічні інструкції

Для деталей по конкретному плагіну — див. нижньорівневий `CLAUDE.md`:
- [`ua/CLAUDE.md`](./ua/CLAUDE.md) — архітектура, агенти, скіли українського плагіна.
- [`pl/CLAUDE.md`](./pl/CLAUDE.md) — те саме для польського плагіна.

## Спільні правила для всіх плагінів

- **Дослівність цитат НПА.** Норма наводиться в точній редакції, чинній на дату, з посиланням на первинне джерело (для UA — zakon.rada.gov.ua; для PL — isap.sejm.gov.pl).
- **Не вигадувати судову практику.** Номери справ, дати, цитати — з офіційних реєстрів (для UA — ЄДРСР, ВС; для PL — Portal Orzeczeń).
- **Плейсхолдери для персональних даних** — не зберігати реальні дані клієнтів у файлах проєкту.
- **Документи — чернетки.** Фінальна редакція і відповідальність — на людині-юристі.
