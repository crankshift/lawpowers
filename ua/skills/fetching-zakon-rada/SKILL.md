# fetching-zakon-rada

Офіційний портал ВРУ — єдине авторитетне джерело текстів НПА України. Правильний URL дозволяє отримати чинну редакцію або редакцію на конкретну історичну дату.

## URL patterns

| Ціль | URL |
|---|---|
| Чинна редакція | `https://zakon.rada.gov.ua/laws/show/{ID}` |
| Редакція на дату | `https://zakon.rada.gov.ua/laws/show/{ID}/ed{YYYYMMDD}` |
| Картка документа | `https://zakon.rada.gov.ua/laws/card/{ID}` |
| Англомовна версія (де є) | `https://zakon.rada.gov.ua/laws/show/en/{ID}` |

де `{ID}` — ідентифікатор документа на zakon.rada.

## Workflow

1. **Визначити ID** з таблиці або WebSearch за назвою НПА.
2. **Чинна редакція** — WebFetch `/laws/show/{ID}`. Перевірити дату редакції у верхній частині сторінки.
3. **Історична редакція** (для правовідносин, що виникли в минулому) — додати `/ed{YYYYMMDD}`. Приклад: `/laws/show/435-15/ed20180101` — ЦК станом на 01.01.2018.
4. **Зміни після шуканої дати** — перевірити блок «Редакції» / «Зв'язки».
5. **Цитування** — стаття + редакція (дата) + URL.

## Common mistakes

- **Посилання без редакції.** Норма могла змінитися — без дати цитата ненадійна.
- **Застарілий фетч.** Редакція могла оновитися між сесіями; перед подачею документа — перевірити свіжо.
- **Агрегатори як primary source.** ligazakon, protocol.ua, judge.ua — лише для навігації; авторитетне джерело — zakon.rada.
- **Ігнорування перехідних положень.** Зміни можуть мати відкладену дію або зберігати попередню редакцію для окремих категорій правовідносин.
- **Плутання ID при подібних законах.** Перевіряти повну назву НПА у картці, не покладатися лише на ID з пам'яті.
