# reviewing-b2b-service-contract

Договір про надання послуг у B2B-обороті (договір співпраці, рамковий договір, IT-контракт, staff augmentation, аутсорсинг) — договір, регульований Главою 63 ЦК (ст. 901–907) та загальним принципом свободи договору (ст. 627 ЦК). Проблема не у формі (проста письмова достатня), а в обмежувальних застереженнях: заборона конкуренції, ексклюзивність, штрафні санкції, ІВ. Цей скіл — чек-лист аудиту та перелік red flags для таких договорів, зокрема для ФОП (фізичної особи-підприємця).

## Сфера застосування

| Тип договору | Характер | Приклади |
|---|---|---|
| Договір про надання послуг (ст. 901-907 ЦК) | Надання за завданням замовника | Консалтинг, програмування, маркетинг |
| Рамковий договір (ст. 627 ЦК) | Рамковий + окремі замовлення/додатки | Framework agreement + замовлення |
| Staff augmentation / body leasing | Надання спеціаліста | ІТ, інжиніринг, фінанси |
| IT-аутсорсинг (T&M / fixed price) | Комплексне надання | Development, maintenance, DevOps |
| Managed services | Утримання / SLA | Інфраструктура, helpdesk, моніторинг |

## Заборона конкуренції в B2B

### Правова основа

Заборона конкуренції в договорі між підприємцями (ФОП ↔ юр. особа, юр. особа ↔ юр. особа) **не** регулюється КЗпП — Кодекс законів про працю стосується виключно трудових відносин. У B2B-обороті обмежувальна клаузула оцінюється через:

1. **Ст. 42 Конституції України** — кожен має право на підприємницьку діяльність, яка не заборонена законом. Обмеження — лише відповідно до закону.
2. **ЦК ст. 627** — свобода договору: сторони вільні в укладенні, але зміст не може суперечити актам цивільного законодавства, моральним засадам суспільства.
3. **ЦК ст. 203** — зміст правочину не може суперечити ЦК, іншим актам, інтересам держави і суспільства, моральним засадам.
4. **ЦК ст. 215 + ст. 228** — правочин, що порушує публічний порядок або суперечить інтересам держави і суспільства — **нікчемний** (недійсний з моменту укладення, без рішення суду).
5. **ГК ст. 42** — підприємництво: самостійна, ініціативна, на власний ризик діяльність з метою одержання прибутку. Обмеження підприємницької діяльності — лише відповідно до ГК та інших законів.

Клаузула, що фактично **позбавляє ФОП можливості здійснювати підприємницьку діяльність** (тобто працювати у своїй галузі взагалі), суперечить ст. 42 Конституції та є **нікчемною**.

### Тест дійсності клаузули

- [ ] **Обсяг визначено конкретно?** Названі конкуренти / перелік клієнтів — не «будь-яка компанія з подібною діяльністю» чи «галузь ІТ». Клаузула, що охоплює всю галузь = потенційно нікчемна (ст. 42 Конституції, ст. 203 ЦК).
- [ ] **Строк пропорційний?** 6–12 місяців після закінчення = прийнятно при обґрунтуванні. Безстрокова заборона = нікчемна (ст. 203, 228 ЦК).
- [ ] **Географічний обсяг обмежений?** Україна / конкретні міста — не «увесь світ».
- [ ] **Предметний обсяг обмежений?** Конкретна технологія / сегмент — не «усі ІТ-послуги».
- [ ] **Компенсація за заборону після закінчення?** Українське законодавство прямо не регулює компенсацію за B2B non-compete, але її відсутність суттєво послаблює обґрунтованість обмеження.
- [ ] **Штрафні санкції пропорційні?** Штраф > 24x місячної оплати = ризик зменшення судом (ст. 551 ЦК).
- [ ] **Не позбавляє ФОП ринку?** Якщо заборона охоплює всі КВЕД ФОП — фактично припиняє діяльність = суперечить ст. 42 Конституції.
- [ ] **Обґрунтований інтерес?** Заборона має захищати конкретний інтерес (комерційна таємниця, база клієнтів) — не слугувати лише обмеженню конкуренції (ст. 42 ГК).

### Судова практика ВС

Судова практика Верховного Суду щодо заборони конкуренції у B2B-договорах в Україні є **менш розвиненою**, ніж у Польщі. Основні правові позиції формуються через:
- Рішення КСУ щодо ст. 42 Конституції (свобода підприємницької діяльності).
- Постанови ВС/КГС щодо штрафних санкцій і їх зменшення (ст. 551 ЦК).
- Постанови ВС щодо нікчемності правочинів (ст. 203, 215, 228 ЦК).

⚠ **Увага**: перед посиланням у клієнтських документах — перевірити конкретні постанови через скіл `ua:searching-edrsr`. Не вигадувати номери справ.

### Відсутність компенсації

На відміну від трудового права (де обмеження конкуренції після звільнення не передбачено взагалі — КЗпП не містить аналогу польської ст. 101² KP), у B2B-обороті сторони вільні встановлювати заборону конкуренції після закінчення договору. Однак відсутність будь-якої компенсації за таку заборону, у поєднанні з широким обсягом, підсилює аргументи про суперечність ст. 203 та ст. 228 ЦК (порушення моральних засад суспільства, невідповідність принципу справедливості).

## Ексклюзивність та прихована трудова

### Ознаки трудових відносин (ст. 21 КЗпП)

| Ознака | Опис | Класифікація |
|---|---|---|
| Заборона працювати на інших замовників | Ексклюзивність = підпорядкування | **КРИТИЧНА** |
| Фіксований графік, визначений замовником | Часове підпорядкування | СУТТЄВА |
| Єдиний замовник тривалий час | Економічна залежність | СУТТЄВА |
| Обов'язок звітувати про інших замовників | Контроль над свободою діяльності | СУТТЄВА |
| Робота на території замовника під наглядом | Організаційне підпорядкування | СУТТЄВА |
| Замовник надає обладнання (ноутбук, ліцензії) | Ознака роботодавця | СУТТЄВА |
| Обов'язок особистого виконання (без субпідряду) | Особистий характер = ознака трудових | СУТТЄВА |
| Фіксована місячна ставка незалежно від результатів | Оплата як за працю | БАЖАНА |

### Ризик кумулятивний

Наявність однієї ознаки = низький ризик. Одночасна наявність **3 і більше** ознак = високий ризик переквалікації. ДПС (Державна податкова служба) та Держпраці можуть ініціювати перевірку. Позов про встановлення факту трудових відносин може подати працівник або Держпраці.

### Наслідки переквалікації

- Ретроактивний ЄСВ (22% від заробітної плати) — за увесь період «B2B» — обтяжує замовника (визнаного роботодавцем) з пенею.
- ПДФО — роботодавець зобов'язаний утримувати й перераховувати як податковий агент.
- Військовий збір — аналогічно.
- Працівник набуває прав: відпустка, лікарняний, надурочні, захист від звільнення.
- Штрафи Держпраці за допуск до роботи без оформлення (ст. 265 КЗпП) — до 10 МЗП за кожного працівника.

## Штрафні санкції (ГК ст. 231, ЦК ст. 549-552)

- **ЦК ст. 549** — неустойка: грошова сума, яку боржник зобов'язаний сплатити у разі порушення. Включає **штраф** (одноразова сума) і **пеню** (нараховується за кожен день/тиждень прострочення).
- **ЦК ст. 550** — право на стягнення збитків понад неустойку, якщо це передбачено договором.
- **ЦК ст. 551** — **зменшення судом**: суд вправі зменшити розмір неустойки, якщо вона значно перевищує розмір заподіяних збитків та за наявності інших обставин, що мають істотне значення.
- **ГК ст. 231** — законні розміри штрафних санкцій: за порушення господарських зобов'язань штрафні санкції можуть бути встановлені законом або договором.
- Заборона конкуренції = немайнове зобов'язання → штраф допустимий.

Орієнтовна оцінка пропорційності:

| Штраф (відносно місячної оплати) | Оцінка |
|---|---|
| ≤ 6x | Низька — скоріше пропорційна |
| 6x–12x | Помірна — прийнятна при обґрунтованому інтересі |
| 12x–24x | Висока — ризик зменшення (ст. 551 ЦК) |
| > 24x | Дуже висока — значний ризик зменшення |

⚠ Шкала **орієнтовна**. Суд оцінює пропорційність in casu.

## ФОП — особливі питання

- **Ст. 42 Конституції**: підприємницька діяльність — конституційне право. Обмеження — лише законом. Договірна клаузула — не закон.
- **ГК ст. 128**: ФОП — суб'єкт господарювання. Має право самостійно визначати порядок здійснення діяльності.
- **КВЕД**: ФОП має зареєстровані КВЕД. Заборона конкуренції, що охоплює **всі** КВЕД ФОП = фактичне припинення діяльності = нікчемна.
- **Єдинозамовність**: ФОП з одним замовником + ексклюзивність + фіксовані години + відсутність субпідряду = повний профіль прихованих трудових відносин.
- **Дія Сіті**: для IT-компаній — резидентів Дія Сіті (ЗУ № 1667-IX) передбачено спеціальний режим гіг-контрактів (ст. 18-21 ЗУ). Гіг-контракт — НЕ трудовий договір, але передбачає спеціальні гарантії (мінімальна оплата, компенсації). Якщо замовник є резидентом Дія Сіті — перевірити, чи застосовується режим гіг-контракту.
- **Група єдиного податку**: ФОП 3-ї групи (5% / 3% з ПДВ) має дохідний ліміт. Обмеження кількості замовників = обмеження доходу = суперечить праву на підприємницьку діяльність.

## Інтелектуальна власність (ІВ)

- **ЗУ «Про авторське право і суміжні права»** (3792-12, нова редакція ЗУ 2811-IX від 01.01.2023) — твір: результат творчої діяльності (код, документація, архітектура, дизайн).
- **Службовий твір (ст. 16)**: майнові права — у роботодавця, але **лише** у трудових відносинах. У B2B — ст. 16 **не застосовується**; права залишаються у виконавця (ФОП), якщо договір не передбачає інше.
- **Передача прав**: вимагає **письмового договору** з визначенням способів використання (ст. 31, 33 ЗУ). Без письмової форми — передача недійсна.
- **Red flags в клаузулах ІВ**:
  - «Усі права на результати, створені під час дії договору» — **КРИТИЧНА**: охоплює приватні проєкти, open source.
  - Передача без визначення способів використання — **недійсна** (ст. 33 ЗУ).
  - Відмова від немайнових прав — **неможлива** (немайнові права автора невідчужувані).

## Red flags — аудит B2B-договору

| Загроза | Ознака | Класифікація | Що робити |
|---|---|---|---|
| **Повна ексклюзивність** | «Виконавець зобов'язується не надавати послуг жодному іншому замовнику» | **КРИТИЧНА** | Ризик переквалікації; перенегоціювати або відмовитися |
| **Заборона на всю галузь** | «в галузі ІТ» без уточнень | **КРИТИЧНА** | Потенційно нікчемна (ст. 42 Конституції); запропонувати конкретний перелік |
| **Безстрокова заборона** | Без дати закінчення, без максимального строку | **КРИТИЧНА** | Нікчемна (ст. 203 ЦК); додати строк |
| **Заборона без компенсації (після закінчення)** | Заборона діє після розірвання, 0 грн | СУТТЄВА | Послаблює виконуваність; запропонувати компенсацію |
| **Штраф > 24x місячної оплати** | Непропорційний штраф | СУТТЄВА | Суд може зменшити (ст. 551 ЦК); торгуватися |
| **ІВ на попередні роботи** | «Усі права на результати, у т.ч. раніше розпочаті» | **КРИТИЧНА** | Звузити до deliverables за договором; виключити pre-existing IP |
| **Одностороннє розірвання** | Замовник — негайно; ФОП — 3 місяці | СУТТЄВА | Зрівняти строки |
| **Відсутність предмету** | «Послуги за домовленістю» | СУТТЄВА | Ризик відсутності істотних умов (ст. 638 ЦК) |
| **Фіксований графік + єдине місце** | «09:00-18:00 в офісі Замовника» | СУТТЄВА | Ознака прихованої трудової |
| **Контроль методології** | «Відповідно до внутрішніх процедур Замовника» | СУТТЄВА | Ознака підпорядкування |
| **Відсутність обмеження відповідальності** | Безмежна відповідальність ФОП | СУТТЄВА | Запропонувати cap (напр., 12x місячної оплати) |
| **Обов'язок інформувати про замовників** | «Виконавець повідомить про кожну іншу співпрацю» | СУТТЄВА | Обмежує свободу ФОП; ознака контролю |

## Обов'язкові елементи договору

| Елемент | Зауваження |
|---|---|
| **Дані сторін** | ФОП: ПІБ, РНОКПП, адреса, дані запису в ЄДР; юр. особа: повна назва, ЄДРПОУ, юр. адреса, ПІБ представника, підстава повноважень |
| **Предмет договору** | Конкретний обсяг послуг (КВЕД, технічний опис); уникати «послуги ІТ» |
| **Ціна** | Ставка (погодинна / місячна / проєктна), валюта (в Україні — гривня, ч. 1 ст. 524 ЦК; еквівалент у валюті — ч. 2 ст. 524), ПДВ, строк оплати |
| **Строк** | Дата початку, закінчення, умови пролонгації, строки розірвання |
| **Звітування / приймання** | Як звітуються години/deliverables, хто приймає, строк приймання, мовчазне прийняття |
| **ІВ** | Передача / ліцензія; способи використання (ст. 31, 33 ЗУ); pre-existing IP; письмова форма |
| **Конфіденційність** | Обсяг, строк, повернення матеріалів |
| **Заборона конкуренції** | Якщо є — з елементами тесту дійсності (розд. 4) |
| **Штрафні санкції** | Штраф / пеня; суми; застереження щодо зменшення |
| **Відповідальність** | Обмеження (cap), виключення, страхування ЦВ |
| **Розірвання** | Одностороння відмова (обидві сторони), наслідки, розрахунок за незавершені роботи |
| **Персональні дані** | Договір про обробку ПД (ЗУ «Про захист персональних даних» 2297-VI) |
| **Підписи** | Форма: письмова / електронна (КЕП) |

## Шаблон звіту аудиту

## Воєнний стан — особливості

- Воєнний стан може впливати на виконання B2B-договорів: бойові дії, евакуація, мобілізація виконавця.
- Форс-мажор — потрібне свідоцтво ТПП (ст. 14-1 ЗУ «Про ТПП»). Саме введення ВС автоматично не звільняє від зобов'язань.
- Мобілізація ФОП-виконавця — підстава для призупинення, але не автоматичне розірвання.
- Детальніше — скіл **[`ua:checking-martial-law-overrides`](../checking-martial-law-overrides/SKILL.md)**.

## Коли цей скіл доповнюється агентом / іншим скілом

- Для повного складання B2B-договору — агент `ua:contract-drafter` (режим складання).
- Для перевірки контрагента в судових реєстрах — скіл `ua:searching-edrsr`.
- Для воєнних обмежень — скіл `ua:checking-martial-law-overrides`.
- Для спору щодо договору — агент `ua:claim-drafter`.
- Для стягнення заборгованості — агент `ua:debt-collector`.
- Для перевірки підсудності (ФОП = господарський суд) — скіл `ua:determining-ua-jurisdiction`.
